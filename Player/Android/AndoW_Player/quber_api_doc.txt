7.14. USB 카메라 

7.14.1. 유의사항 

USB 카메라는 다중 카메라 사용을 보장하지 않으며, USB 카메라 성능에 따라 사진 또는 영상 품질에 

편차가 있을 수 있다. 

USB 카메라 사용을 위해선 아래 절차대로 수행해야 한다. 

1.  사진 촬영 

1.1  카메라 목록 읽기 

1.2  사진 촬영  

2.  동영상 촬영 

2.1  카메라 목록 읽기 

2.2  동영상 촬영 

2.3  동영상 종료 

카메라 사용하기전 카메라 목록 읽어 검색된 카메라 아이디(cameraId)를 이용해 사진 및 동영상 촬영을

2.3  바코드 읽기 종료 

바코드 읽기 사용하기전 카메라 목록 읽어 검색된 카메라 아이디(cameraId)를 이용해 바코드 

읽기를 요청해야 한다. 카메라 아이디(cameraId)는 장치 상태에 따라 변경될 수 있기 때문에 

바코드 사용 시 매번 읽어 사용하길 권장한다. 

2.  바코드 읽기는 설치 USB 카메라 성능에 따라 처리 속도 및 인식률 차이가 있을 수 있다. 

3.  바코드 읽기 시 밝기, 바코드와 거리, 바코드 크기 등 주변 환경에 영향을 받아 인식률 차이가 

있을 수 있다. 

4.  바코드 읽기 결과는 주변 환경에 따라 데이터 손실이 발생될 수 있다. 

5.  바코드가 코팅된 용지나 곡선면에 출력된 경우 인식률이 떨어질 수 있다.

6.  USB 카메라와 10Cm 이내 거리에서 Linear Codes 인식률 높이기 위해선 바코드 형식에 따라 

차이가 있지만 4 X 1 Cm (가로 X 세로) 이상 바코드 사용을 권장한다. 

7.  USB 카메라와 10Cm 이내 거리에서 2D Codes 인식률 높이기 위해선 바코드 형식에 따라 차이가 

있지만 1 X 1 Cm (가로 X 세로) 이상 바코드 사용을 권장한다. 

8.  바코드 한번 읽기는 요청 후 10 초이내 인식되지 않을 경우 타임오버 응답을 받는다. 

9.  바코드 계속 읽기 시작은 요청 후 계속 읽기 종료 요청까지 바코드 읽기 준비 상태로 되어 있어 

바코드를 추가 읽기 요청 없이 계속 읽을 수 있다. 

10.  바코드를 읽은 준비가 된 경우 “삐삐” , 바코드 데이터를 읽은 경우 “삐”, 바코드 읽기 타이오버 

발생된 경우 “삐리링” 사운드가 스피커를 통해 발생된다.

8.4. Open API 요청 및 응답 규칙 

8.4.1. 요청 규칙 

  사이니지의 상태 또는 설정 요청 시 AIDL 을 이용해 규격화된 JSON 메시지를 이용해야 한다. 

  동시 명령 요청 처리를 위한 버퍼가 존재하며 기본 버퍼 크기는 30 개이다. 

  요청 시 메시지 구분 및 비동기 처리를 위해 명령 코드(cmdCode)와 요청 아이디(requestId) 

정보를 필수 항목으로 포함해야 한다. 

  요청 아이디(requestId)는 고객 Apps 에서 구분할 수 있는 임의의 값을 직접 생성하고 관리해야 

한다. 

형식 예: yyyyMMddHHmmssSSS 

       값: 20210210143542001 

  요청 아이디(requestId)에 특수 문자 ‘@’는 사용할 수 없다. 

  기 요청한 요청 명령의 응답 수신 없이 동일 명령 코드 (cmdCode) 를 연속 요청하는 것을 

허용하지 않는다. 

  명령 코드(cmdCode)는 “8.3.2 명령 코드”를 참고해 사용한다.

8.4.2. 응답 규칙 

  명령 요청에 대한 응답은 JSON 메시지 형식으로 한다. 

  성공 응답 시 명령 요청 처리 결과 코드 값을 응답 메시지에 포함한다. 

  성공 응답 시 메시지 구분 및 비동기 처리를 위해 요청 명령 코드(cmdCode)를 응답항목에 

포함하고 요청 아이디(requestId) 값을 응답 아이디(responseId) 값으로 반환한다. 

  실패 응답 시 명령 코드 값이 없는 경우도 있다. 

  실패 응답 시 요청 아이디 값과 응답 아이디 값이 다를 수 있다. 

  요청 기능에 따라 응답 수신 전 사이니지가 재부팅 할 수 있다. 

  예외가 발생할 경우 응답하지 않을 수 있다.
